# tz-data 前端菜单导航规范

> 基于交易日历需求，梳理当前页面与菜单体系，规划新的菜单导航结构。

## 一、当前菜单结构（现状）

```
侧边栏菜单（按 group 分组）
├── 数据维护
│   ├── 数据维护看板    /dashboard
│   ├── 数据目录        /catalogs
│   ├── 同步任务        /sync-tasks
│   └── 健康快照        /health-snapshots
├── 基础数据
│   ├── 交易所管理      /exchanges
│   └── 品种管理        /products
├── 交易日历
│   ├── 交易日历        /trade-calendar      （4个Tab：交易所日历/日历浏览/列表查询/商品日历）
│   ├── 商品日历        /product-calendar     （复用 TradeCalendarList.vue，自动切换到商品Tab）
│   └── 合约管理        /contracts
├── 账单与账户
│   ├── 账户管理        /accounts
│   └── 账单管理        /statements
└── 系统
    ├── 数据源配置      /data-sources
    └── 告警历史        /alerts
```

### 现状问题

1. **交易日历页面功能过载**：一个页面承载 4 个 Tab（交易所日历、日历浏览、列表查询、商品日历），职责混杂
2. **路由与 Tab 重复**：`/trade-calendar` 和 `/product-calendar` 路由区分仅用于自动切换 Tab，但页面完全复用
3. **菜单分组不够清晰**：交易日历相关的"合约管理"与"品种管理"分散在不同分组
4. **缺少合约同步入口**：合约导入、到期检查等功能没有独立菜单入口

## 二、目标菜单结构

```
侧边栏菜单（按 group 分组，order 排序）
├── 数据维护                    ← 现有不变
│   ├── 数据维护看板    /dashboard           order: 10
│   ├── 数据目录        /catalogs            order: 20
│   ├── 同步任务        /sync-tasks          order: 30
│   └── 健康快照        /health-snapshots    order: 40
│
├── 基础数据                    ← 扩展
│   ├── 交易所管理      /exchanges           order: 50
│   ├── 品种管理        /products            order: 60
│   └── 合约管理        /contracts           order: 70  ← 从"交易日历"移入
│
├── 交易日历                    ← 重组，每个功能独立页面
│   ├── 日历概览        /trade-calendar      order: 80   ← 交易所日历状态 + 产品概览（原Tab1）
│   ├── 日历浏览        /calendar-browse     order: 85   ← 月历/列表查看（原Tab2+Tab3合并）
│   └── 商品日历        /product-calendar    order: 90   ← 商品维度日历（原Tab4独立）
│
├── 账单与账户                  ← 现有不变
│   ├── 账户管理        /accounts            order: 100
│   └── 账单管理        /statements          order: 110
│
└── 系统                        ← 现有不变
    ├── 数据源配置      /data-sources        order: 120
    └── 告警历史        /alerts              order: 130
```

### 变更说明

| 变更项 | 旧 | 新 | 说明 |
|--------|-----|-----|------|
| 合约管理 | group=交易日历 | group=基础数据 | 合约属于基础数据模型 |
| 日历浏览 | Tab（/trade-calendar内） | 独立页面 /calendar-browse | 将日历浏览+列表查询拆为独立页面 |
| 商品日历 | Tab（/trade-calendar内）+ 独立路由复用 | 独立页面 /product-calendar | 独立组件，不再复用 TradeCalendarList |
| 日历概览 | Tab（/trade-calendar内） | /trade-calendar 主页面 | 保留为交易日历首页，展示系统状态和产品概览 |

  component: () => import('@/views/PageComponent.vue'),
  meta: {
    title: '页面标题',       // 侧边栏显示名称

## 五、菜单分组规则

### 5.1 分组定义

| 分组 | 职责 | 包含页面 |
|------|------|----------|
| **数据维护** | 数据同步、监控、健康检查 | 看板、目录、同步任务、快照 |
| **基础数据** | 静态配置数据管理 | 交易所、品种、合约 |
| **交易日历** | 交易日判定、日历展示 | 概览、浏览、商品日历 |
| **账单与账户** | 用户财务数据 | 账户、账单 |
| **系统** | 系统配置与运维 | 数据源、告警 |

### 5.2 分组排序

分组按在路由表中**首次出现的 order 值**排序，无需额外配置。同一分组内的页面按各自 order 排序。

### 5.3 侧边栏渲染逻辑

当前侧边栏（MainLayout.vue）使用以下逻辑：

```javascript
// 从路由中提取有 meta.title 的子路由，按 order 排序
const menuRoutes = computed(() => {
  const routes = route.matched[0]?.children?.filter(r => r.meta && r.meta.title) || []
  return routes.sort((a, b) => (a.meta.order || 999) - (b.meta.order || 999))
})

// 连续相同 group 的路由归为一组
const groupedMenuRoutes = computed(() => {
  const groups = []
  let currentGroup = null
  for (const r of menuRoutes.value) {
    const groupName = r.meta.group || '其他'
    if (!currentGroup || currentGroup.name !== groupName) {
      currentGroup = { name: groupName, items: [] }
      groups.push(currentGroup)
    }
    currentGroup.items.push(r)
  }
  return groups
})
```

**关键约束**：同一分组的页面在路由数组中必须**连续排列**，否则会被分成多个同名分组。
