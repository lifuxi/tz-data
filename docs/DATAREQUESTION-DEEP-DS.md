# 数据维护与同步系统设计方案（完整版）

> **整合说明**：本方案融合了之前设计的“数据维护看板”具体功能与用户提供的“数据维护与同步系统架构”层次结构，形成从数据源到应用层的完整技术方案。涵盖交易所数据、第三方数据、本地数据、账单数据等全类型数据的统一管理、状态监控、差异检测与一键补齐。

---

## 一、总体架构

## 二、数据源层详细设计

### 2.1 交易所数据源

支持国内四大期货交易所的数据接入：

| 交易所 | 数据内容 | 接入方式 | 更新频率 |
|--------|----------|----------|----------|
| 中金所 (CFFEX) | 股指期货/期权日线、分钟线、合约信息、机构持仓前20名 | 官方API / 爬虫 | 每日 |
| 上期所 (SHFE) | 商品期货/期权日线、合约信息、机构持仓 | 官方API / 第三方 | 每日 |
| 大商所 (DCE) | 商品期货/期权日线、合约信息 | 第三方（Tushare） | 每日 |
| 郑商所 (CZCE) | 商品期货/期权日线、合约信息 | 第三方（Tushare） | 每日 |

### 2.2 第三方数据源

| 数据源 | 用途 | 获取方式 | 优先级 |
|--------|------|----------|--------|
| Wind | 高质量行情、宏观数据 | API（需购买） | 主源（若可用） |
| 同花顺 | 行情、机构持仓 | API/爬虫 | 备源 |
| 聚宽 | 基础行情数据 | API（免费/付费） | 备源 |
| Tushare | 期权、期货日线/分钟线 | API（积分制） | 主源（个人使用） |

### 2.3 本地数据源

| 类型 | 内容 | 存储位置 |
|------|------|----------|
| 数据库 | 用户自行导入的历史数据 | PostgreSQL / ClickHouse |
| 文件存储 | 期货公司日结账单（CSV/Excel）、离线数据包 | 本地磁盘或云存储 |

## 三、数据管理层详细设计

### 3.1 元数据管理

负责维护系统内所有可跟踪的数据目录实体。

| 功能模块 | 说明 | 数据表 |
|----------|------|--------|
| 交易所管理 | 维护支持的交易所列表、交易时段、时区 | `exchange_config` |
| 品种管理 | 维护品种代码、名称、所属交易所、品种类型 | `product_config` |
| 合约管理 | 维护具体合约（如IF2506）的上市日、到期日、乘数等 | `contract_info` |
| 数据目录配置 | 用户或系统预设需要跟踪的数据项（每个品种+数据类型组合） | `data_catalog` |

**前端交互**：用户可在“数据源管理”页面中，通过界面添加/删除需要跟踪的数据项（如“中金所-中证1000指数-分钟线”）。

### 3.2 数据状态监控

#### 3.2.1 数据完整性检查

- 每日自动对比本地数据与交易日历，检查是否存在缺失日期的数据。
- 对于每个数据项，记录本地最新日期、总记录数。
- 标注“部分缺失”或“完整”。

#### 3.2.2 数据质量评估

| 指标 | 检查内容 | 阈值 |
|------|----------|------|
| 缺失率 | 缺失日期数 / 应有日期数 | >5% 告警 |
| 异常值 | 价格或持仓量 <0 或超大 | 产生告警 |
| 一致性 | 多数据源对比差异 | 收盘价误差 >0.5% 告警 |

#### 3.2.3 差异对比引擎

- 获取本地最新日期、远程最新日期（通过数据源适配器查询或交易日历推断）。
- 计算缺失日期范围。
- 支持按交易所、品种、数据类型筛选差异结果。
- 差异结果存入 `data_status_local` 和 `data_status_remote` 表。

### 3.3 数据同步引擎

| 功能 | 说明 |
|------|------|
| 增量同步 | 仅拉取本地缺失日期范围的数据，默认模式，每日定时执行 |
| 全量同步 | 忽略本地已有数据，重新拉取指定时间范围的全量数据（用于初始化或修复） |
| 断点续传 | 当同步任务因网络或限流中断时，记录已成功拉取的最后日期；下次继续从断点处开始，避免重复 |

**同步流程**：
1. 根据数据源适配器，分批请求（如每30天一批）。
2. 每批数据写入数据库前进行校验。
3. 若某批失败，记录失败点并重试（最多3次），仍失败则暂停并告警。
4. 所有批次完成后更新 `data_status_local`。

## 四、任务调度层详细设计

### 4.1 定时任务调度

| 任务名称 | 执行时间 | 描述 |
|----------|----------|------|
| 每日行情增量同步 | 每日 18:00 | 对每个启用的数据项，拉取最新交易日数据 |
| 每日状态刷新 | 每日 18:30 | 刷新本地最新日期、远程最新日期 |
| 数据完整性检查 | 每日 19:00 | 检查缺失数据，生成差异报告并告警 |
| 账单缺失检测 | 每日 20:00 | 检查各期货账户缺失账单日期，发送提醒 |

### 4.2 手动任务触发

- 用户通过 Web 界面点击“同步”按钮，触发指定数据项的同步任务。
- 支持“全量同步”和“增量同步”模式选择。
- 支持批量选择多个数据项，一次性提交任务队列。

### 4.3 任务队列管理

- 使用 Celery 作为分布式任务队列。
- 任务按优先级排队（高优先级：用户手动触发的补齐；低优先级：定时自动同步）。
- 支持并发控制（如同时最多运行3个数据拉取任务），避免触发数据源限流。

### 4.4 任务状态监控

- 每个任务有唯一 ID，记录开始时间、结束时间、状态（pending/running/success/failed）、拉取记录数、错误信息。
- 前端通过轮询或 WebSocket 实时获取任务进度。
- 失败任务自动重试（最多3次），仍失败则发送 IM 告警。

## 五、存储层详细设计

### 5.1 关系数据库（PostgreSQL）

存储结构化业务数据和元数据：

| 表/模式 | 内容 |
|---------|------|
| `metadata` | 交易所、品种、合约、数据目录配置 |
| `task` | 同步任务记录、任务日志 |
| `status` | 数据状态本地快照、远程快照 |
| `statement` | 账单解析结果（资金流水、交易明细、持仓汇总） |
| `user` | 用户信息、期货账户、监控中心凭证（加密） |

### 5.2 时序数据库（ClickHouse）

存储海量行情数据，优化时间范围查询：

| 表 | 字段 | 分区 |
|----|------|------|
| `index_minute` | 指数分钟线 | 按日期分区 |
| `future_minute` | 期货分钟线 | 按日期分区 |
| `option_minute` | 期权分钟线（含IV） | 按日期分区 |
| `top20_holdings` | 机构持仓历史 | 按日期分区 |

> **注**：如资源有限，可先用 QuestDB 替代 ClickHouse（两者均为列式时序数据库）。

### 5.3 缓存层（Redis）

用于提升查询性能和任务状态同步：

- 任务状态缓存：存储正在运行的任务进度，供前端轮询。
- 数据状态缓存：存储每个数据项的本地/远程最新日期差异结果，减少数据库查询。
- 会话缓存：用户登录状态、设置。

## 六、账单数据管理模块（独立子系统）

账单数据管理作为数据管理层之上的一个独立模块，与通用数据同步引擎并行。其架构如下：

```
┌─────────────────────────────────────────────────────────────────────┐
│                        账单数据管理模块                              │
├─────────────────────────────────────────────────────────────────────┤
│ 账单源配置管理                                                       │
│ ├── 期货公司账户管理（多账户支持，账户名称、账号、交易所权限）         │
│ ├── 登录凭证安全存储（AES-256加密存储监控中心密码、API密钥）         │
│ └── 账单格式适配器（支持不同期货公司账单CSV/Excel模板解析）           │
├─────────────────────────────────────────────────────────────────────┤
│ 账单完整性监控                                                       │
│ ├── 每日自动对账（交易日历 vs 已上传账单日期，生成缺失报告）          │
│ ├── 缺失账单检测（结合交易日历与statement_status表）                │
│ └── 解析失败监控（记录失败行数、错误原因，支持手动修复）              │
├─────────────────────────────────────────────────────────────────────┤
│ 账单数据同步                                                         │
│ ├── 自动化下载（若用户配置监控中心账号，模拟登录自动抓取）            │
│ ├── 智能解析引擎（多格式支持，容错处理）                             │
│ └── 数据质量校验（资金流水平衡校验、交易明细与成交编号一致性）         │
├─────────────────────────────────────────────────────────────────────┤
│ 账单与行情对账                                                       │
│ ├── 成交记录匹配（将交易明细中的成交价与当时市场行情对比）            │
│ ├── 资金流水验证（计算理论资金变化与账单资金差异）                    │
│ └── 差异报告生成（展示滑点、成交延误等执行质量指标）                  │
├─────────────────────────────────────────────────────────────────────┤
│ 账单补全工作流                                                       │
│ ├── 一键补全缺失账单（支持多日、多账户批量补传）                      │
│ ├── 断点续传支持（若自动下载中断，记录最后成功日期，下次继续）         │
│ └── 补全结果报告（成功/失败明细，提供错误行供下载修正）               │
└─────────────────────────────────────────────────────────────────────┘
```

### 6.1 账单源配置管理

- 用户在“数据管理”->“期货账户管理”中配置账户信息。
- 支持多个期货账户（如中信期货账户、永安期货账户）。
- 登录凭证加密存储，仅在自动抓取时临时解密。

### 6.2 账单完整性监控

- 每日定时任务根据交易日历检查每个账户是否有对应的账单解析记录。
- 缺失账单在前端“数据维护看板”中显示，并提供“补齐”按钮。
- 解析失败时，标记部分成功，用户可查看错误行并重新上传。

### 6.3 账单数据同步

- 手动上传：用户选择本地文件（支持单日或多日压缩包），系统解析并入库。
- 自动抓取：用户授权后，系统根据账户凭证登录中国期货监控中心，下载缺失账单。

### 6.4 账单与行情对账

- 将交易明细中的成交价与对应时间点的行情价格对比，计算滑点。
- 生成执行质量报告，供账单分析模块使用。

### 6.5 账单补全工作流

- 在数据维护看板上，每个账户行显示缺失账单天数，点击“一键补全”。
- 系统自动按顺序补齐缺失日期，支持断点续传。
- 补全完成后生成报告，展示成功/失败详情。

## 七、Web管理界面设计

### 7.1 数据维护看板（核心页面）

- 左侧树形结构：按交易所 → 品种 → 数据类型（日线/分钟线/机构持仓/合约信息）组织。
- 右侧表格：展示每个数据项的本地最新日期、远程最新日期、差异天数、最后同步时间。
- 操作列：提供“同步”按钮（支持增量/全量选项）和“详情”按钮（查看缺失日期列表）。
- 账单数据分组：单独展示“期货账单”节点，下挂各账户，展示缺失天数和补齐按钮。

### 7.2 期货账户管理页面

- 列表展示账户ID、名称、状态（活跃/禁用）、最后同步时间。
- 支持新增、编辑、删除账户。
- 支持配置监控中心登录凭证（用户名、密码）。
- 支持测试连接功能。

### 7.3 任务监控页面

- 展示当前运行中的同步任务队列。
- 展示历史任务列表，支持查看详情和重试失败任务。

## 八、API概要设计

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/metadata/exchanges` | GET | 获取交易所列表 |
| `/api/metadata/products?exchange={exchange}` | GET | 获取品种列表 |
| `/api/catalog` | CRUD | 管理数据目录配置 |
| `/api/status/diff` | GET | 获取所有数据项的差异状态 |
| `/api/sync/trigger` | POST | 触发现数据同步任务（参数：catalog_id, mode=incremental/full） |
| `/api/sync/task/{id}` | GET | 查询任务状态 |
| `/api/statement/accounts` | CRUD | 管理期货账户 |
| `/api/statement/upload` | POST | 上传账单文件 |
| `/api/statement/auto_fetch` | POST | 触发自动抓取 |

## 九、扩展性说明

- **新增交易所/品种/数据类型**：只需在元数据表中添加记录，无需修改代码。
- **新增数据源**：实现 `DataAdapter` 接口，注册到数据源工厂即可。
- **新增账单格式**：添加新的解析器实现，在账单格式适配器中注册。

---

本设计方案将用户提供的架构层次与之前详细的功能需求完美融合，形成了一套完整的数据维护与同步系统方案，可直接指导后续开发。
