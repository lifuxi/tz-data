## 一、QuestDB 时序数据库启用

### 1.1 背景

行情数据原全部存储在 SQLite（`tzdata_market.db`），仅支持行式存储，不适合时间序列查询。引入 QuestDB 作为时序数据库，提供高性能时间序列查询与归档能力。

### 1.2 实施内容

| 文件 | 变更 |
|------|------|
| `storage/db_registry.py` | 新增 `get_questdb_connection()`、`close_questdb()`、`init_questdb_schema()` |
| `storage/questdb_store.py` | 完全重写，修复 import、时间戳构建、批量插入 |
| `storage/schemas/questdb.sql` | 补充 `daily_quotes` 表定义 |
| `.env` | 启用 QuestDB 配置（host/port/db） |
| `data/questdb-*/` | 安装 questdb-9.3.5 到工程目录 |

### 1.3 设计要点

- **条件导入**：QuestDB 未启动时不影响核心功能
- **psycopg2 连接**：通过 PostgreSQL wire protocol（端口 8812）
- **Schema 初始化**：通过 HTTP API（端口 9000）执行 DDL
- **双写模式**：下载器同时写入 SQLite + QuestDB，QuestDB 失败静默跳过

### 1.4 迁移结果

| 表 | SQLite 源 | QuestDB 目标 | 状态 |
|---|---|---|---|
| `minute_quotes` → `future_minute` | 720 行 | 720 行 | 一致 |
| `daily_quotes` → `daily_quotes` | 1,034,624 行 | 1,013,987 行 | 一致（20,637 行重复跳过） |

迁移脚本：`migrations/sqlite_to_questdb.py`（支持 dry-run / run / verify）

## 二、多周期 K 线补齐（5min/15min/30min/60min）

### 2.1 重采样引擎

| 文件 | 说明 |
|------|------|
| `maintenance/analysis/resampler.py` | OHLCVResampler 类，pandas resample 聚合 |
| `maintenance/analysis/resample_writer.py` | ResampleWriter 双写（SQLite + QuestDB） |

聚合规则：
- `open` → first, `high` → max, `low` → min, `close` → last
- `volume` → sum, `turnover` → sum, `open_interest` → last

### 2.2 定时任务

| 文件 | 说明 |
|------|------|
| `scheduler/tasks/resample_tasks.py` | Celery 任务，每日 16:00 自动执行 |
| `scheduler/celery_app.py` | beat_schedule 注册 `daily-resample-multi-freq` |

### 2.3 历史回填 CLI

```bash
python -m tzdata_pkg.cli.resample_backfill --freq all      # 所有周期
python -m tzdata_pkg.cli.resample_backfill --freq 5min     # 单周期
python -m tzdata_pkg.cli.resample_backfill --contract IF0  # 指定合约
```

### 2.4 API 查询

新增端点：`GET /api/v2/minute-quotes/frequencies/{contract_code}`
返回某合约已有哪些频率的数据及条形数统计。
