tz-data 工程需求分析与测试覆盖差距
一、业务需求全景图
编号	业务域	核心需求
B1	行情采集	从 CFFEX/SHFE/Tushare/Akshare/CFMMC 5 个数据源采集日线、分钟线、持仓排名、期权 IV/Greeks
B2	数据存储	3 库分离架构（market/trading/analysis），100+ 张 SQLite 表
B3	定时调度	22 个 Celery Beat 任务，按交易日/每日/每周调度
B4	数据同步	增量/全量同步，断点续传，并发控制，审计日志
B5	账单管理	CFMMC 账单解析、上传、自动抓取、资金流水核对、交易配对、盈亏计算
B6	数据质量	健康快照、完整性检查、跨源对比、质量评估、缺失检测
B7	交易日历	交易日历维护、特殊日期覆盖、品种级日历、交易时间模板
B8	合约管理	合约信息同步、主力合约识别、到期检测、连续合约映射
B9	数据查询	50+ REST API，涵盖行情、持仓、账单、信号、机构、期权
B10	告警通知	同步失败、数据缺失、DingTalk Webhook 推送
B11	分析计算	市场环境分类、VWAP 计算、Greeks 预计算、信号生成
B12	配置管理	系统配置、交易所配置、品种配置、数据目录管理
二、非功能需求
编号	维度	要求
N1	数据一致性	3 库之间的外键关系（catalog_id、account_id）必须一致
N2	数据完整性	每个 catalog 的数据覆盖度 ≥ 95%，无空洞
N3	时间服务	交易日历必须 100% 准确，同步调度必须精确到 ±5 分钟
N4	并发安全	SQLite 连接池 WAL 模式，防止并发写冲突
N5	错误恢复	下载失败自动重试（max_retries=3），断点续传
N6	可观测性	审计日志、同步日志、质量报告、DingTalk 告警
N7	性能	API 响应 < 500ms，下载任务 < 1 小时
三、测试未覆盖的关键领域
1. 数据质量 — 最高风险
领域	现状	风险等级	说明
数据量一致性	无测试	🔴 极高	daily_quotes/position_detail 等核心表的记录数是否与交易所实际发布数量一致？无自动化校验
数据空洞检测	仅 completeness_checker.py 有代码，无集成测试	🔴 极高	CFFEX 日线是否每天漏采？分钟线是否缺时段？无每日自动告警验证
跨库数据一致性	verify/cross_db_check.py 存在但无自动化调度	🔴 极高	data_catalog 中的 total_records 与实际表中的记录数是否一致？data_status_local 是否与真实数据一致？
数据格式校验	无测试	🟡 高	日期格式统一性（YYYYMMDD vs YYYY-MM-DD）、价格/结算价的数值范围、volume 非负
重复数据	无测试	🟡 高	同一合约同一日期的数据是否重复插入？download_log 是否有重复记录？
异常值检测	无测试	🟡 高	涨跌幅 > 100%、价格为 0、volume 突增 10 倍等异常数据
建议新增：

每日定时任务：数据量对账（catalog total_records vs 实际 COUNT(*)）
每日定时任务：数据空洞检查（交易日历 vs 实际有数据的日期）
单元测试：价格范围、日期格式、数值合法性校验
集成测试：completeness_checker.py 的端到端验证
2. 数据量/数量 — 高风险
领域	现状	风险等级	说明
全量数据回溯	无测试	🔴 高	每个 catalog 的 earliest_date 到 latest_date 是否连续？是否有断档？
增量同步幂等	sync_engine.py 有代码，无测试	🔴 高	同一天增量同步多次是否产生重复记录？
分钟数据覆盖度	minute_completeness.py 存在，无测试	🔴 高	MO 分钟线是否覆盖全部交易时段（9:00-15:00, 21:00-02:30）？
持仓数据连续性	无测试	🟡 高	position_detail 是否有缺失交易日？Top20 是否每天都抓到了？
账单数据完整性	bill_tasks.py 有日历检查，无测试	🟡 高	账单日期是否覆盖所有交易日？缺失账单是否已触发告警？
建议新增：

测试：增量同步幂等性（同一日期同步 2 次，记录数不变）
测试：分钟数据时段覆盖度验证
测试：持仓数据连续交易日验证
测试：账单日历缺失告警触发
3. 时间服务 — 高风险
领域	现状	风险等级	说明
交易日历准确性	有单元测试，无端到端集成测试	🔴 高	2026 年的交易日是否与交易所官方日历一致？节假日/调休是否正确？
定时调度执行时间	无监控	🟡 高	22 个 Beat 任务是否都在预期时间执行？延迟/遗漏如何检测？
夜盘跨日处理	无测试	🔴 高	周一 21:00 夜盘数据归属周五还是周一？跨日合约日期处理是否正确？
时区处理	无测试	🟡 中	Celery timezone=Asia/Shanghai，但 SQLite 存储的是 UTC 还是本地时间？
特殊日期覆盖	special_date_override 表存在，无测试	🟡 中	交易所临时调整交易日是否被正确处理？
建议新增：

测试：交易日历 vs 交易所官方日历对比（年度验证）
测试：夜盘跨日场景（周五夜盘 → 周一交易日归属）
测试：Celery Beat 调度时间偏差检测
测试：特殊日期覆盖的优先级
4. 账单与交易 — 高风险
领域	现状	风险等级	说明
账单解析准确性	test_bill_fund_flows.py 存在	🟡 中	仅覆盖了资金流水，未覆盖交易记录、持仓记录、手续费计算
FIFO 配对正确性	无测试	🔴 高	复杂场景：部分平仓、多次加仓后平仓、跨日配对
盈亏计算准确性	无测试	🔴 高	不同合约乘数（IF=300, ag=1）下的毛盈亏、净盈亏、手续费计算
CFMMC 自动抓取	cfmmc_scraper.py 存在，无测试	🔴 高	Playwright 爬虫是否能在页面变化时正确检测并告警？
账单解析格式兼容	无测试	🟡 高	不同期货公司的账单格式差异（UTF-8/GBK/GB2312、表格结构）
建议新增：

测试：FIFO 配对 5 种以上复杂场景
测试：盈亏计算（多合约乘数、手续费扣除）
测试：账单解析格式兼容性矩阵
集成测试：CFMMC 爬虫健康检查
